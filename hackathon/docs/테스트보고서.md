## 4. E2E 초기 실패 및 수정 이력

최초 실행 시 4개 테스트 실패 → 원인 파악 후 수정, 전체 통과.

| # | 테스트 | 실패 원인 | 수정 내용 |
|---|--------|---------|---------|
| 1 | 잘못된 자격증명으로 로그인 시 에러 메시지 | 정규식 `/잘못\|오류\|실패/i`가 실제 메시지 "아이디 또는 비밀번호가 올바르지 않습니다."와 불일치 | 정규식에 `올바르지` 패턴 추가 |
| 2 | 정렬 드롭다운이 표시된다 | `SortDropdown`이 `<button>`이 아닌 `<select>` 요소로 구현됨 | `getByRole('button')` → `getByRole('combobox')` |
| 3 | 통계 카드가 표시된다 | 여러 요소가 매칭될 때 `toBeVisible()` 단일 요소 요구 | `.first()` 추가 |
| 4 | 게임 추가 모달 필수값 미입력 시 유효성 오류 | 여러 에러 메시지 동시 렌더링 시 다중 매칭으로 `toBeVisible()` 실패 | `.first()` + 명시적 `timeout` 추가 |

```

│  - API 클라이언트 함수                         │
├─────────────────────────────────────────────┤
│  단위 테스트 - 백엔드 (xUnit)                  │
│  - 비즈니스 로직 (Service 레이어)              │
│  - 에지케이스 및 예외 처리                      │
└─────────────────────────────────────────────┘
```

### 테스트 우선순위

| 우선순위 | 대상 | 방법 |
|---------|------|------|
| P1 (필수) | 백엔드 서비스 로직 | xUnit 단위 테스트 |
| P1 (필수) | 핵심 UI 컴포넌트 | Jest + RTL |
| P2 (중요) | 사용자 시나리오 | Playwright E2E |
| P3 (권장) | admin 컴포넌트 | Playwright E2E |
