# 헬스케어 게이미피케이션 큐레이터
## 요구사항 정의서 v1.0

---

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | 헬스케어 게이미피케이션 큐레이터 |
| 목적 | 건강 목표 달성에 도움되는 게임을 AI 기반으로 큐레이션하여 제공 |
| 대상 사용자 | 건강 관심 있는 일반 사용자 |
| 개발 유형 | 풀스택 웹 애플리케이션 |
| 개발 기간 | 해커톤 (1~2일) |

---

## 2. 목표 및 배경

- 증가하는 활동성 헬스케어(게이미피케이션) 트렌드 대응
- 건강에 도움되는 게임을 찾기 어려운 사용자 불편 해소
- AI를 활용해 자동 건강 효과 태깅으로 신뢰성 있는 큐레이션 제공

---

## 3. 사용자 요구사항

### 3.1 일반 사용자

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| U-01 | 건강 카테고리별 게임 목록을 확인할 수 있다 | 상 |
| U-02 | 게임 별 정보(평점, 설명, 다운로드 수)를 볼 수 있다 | 상 |
| U-03 | 태그 목록 및 인기 게임을 확인할 수 있다 | 상 |
| U-04 | 게임별 건강 효과 태그를 확인할 수 있다 | 상 |
| U-05 | 키워드로 게임을 검색할 수 있다 | 상 |
| U-06 | 건강 목표를 입력하면 맞춤 게임을 추천받을 수 있다 | 상 |
| U-07 | 게임 스토어로 바로 이동할 수 있다 | 상 |
| U-08 | 게임 목록을 정렬하고 볼 수 있다 | 상 |

### 3.2 관리자

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| A-01 | 게임 데이터를 관리자로 추가/수정/삭제할 수 있다 | 상 |
| A-02 | AI 태그 결과를 확인하고 수정할 수 있다 | 상 |
| A-03 | 게임 수집(크롤링/API)을 수동으로 트리거할 수 있다 | 상 |

## 4. 기능 요구사항

### 4.1 게임 목록 및 탐색

| ID | 설명 |
|----|------|
| FR-01 | 게임 목록에서 인기 순위 / 태그 필터 / 카테고리별 정렬 기능 제공 |
| FR-02 | 카테고리 목록: 달, 명상·스트레스 해소, 팔 아래, 반응훈련, 밸런스, 파씨지 |
| FR-03 | 정렬 옵션: 인기순, 평점순, 최신순 |
| FR-04 | 게임 카드 표시 항목: 썸네일, 게임명, 카테고리, 평점, 건강 효과 태그 |

### 4.2 게임 상세 페이지

| ID | 설명 |
|----|------|
| FR-05 | 게임 기본 정보 표시 (설명, 개발사, 스크린샷, 평점, 다운로드 수) |
| FR-06 | AI가 분석한 건강 효과 설명 표시 |
| FR-07 | Google Play / App Store 바로가기 링크 제공 |
| FR-08 | 게임의 건강 효과에 따른 유사 게임 추천 (태그 기준) |

### 4.3 AI 건강 효과 태깅

| ID | 설명 |
|----|------|
| FR-09 | 게임 메타데이터에 Claude API를 활용하여 건강 효과 자동 분류 |
| FR-10 | 태그 목록: #심폐기능, #근력강화, #스트레스해소, #인지개선, #반응훈련 |
| FR-11 | 신뢰도와 함께 표시 (AI 분석 근거) |

### 4.4 맞춤 추천

| ID | 설명 |
|----|------|
| FR-12 | 사용자가 건강 목표 선택 (간단한 폼 기반 입력) |
| FR-13 | 선택한 목표에 맞는 게임 리스트 반환 |
| FR-14 | 추천 이유 텍스트 제공 (AI 생성) |

### 4.5 검색

| ID | 설명 |
|----|------|
| FR-15 | 게임명, 건강 효과 태그, 카테고리로 검색 가능 |
| FR-16 | 검색 결과 하이라이팅 |

### 4.6 데이터 수집

| ID | 설명 |
|----|------|
| FR-17 | Google Play API 또는 RapidAPI를 통해 게임 데이터 수집 |
| FR-18 | 주기적 업데이트 기능 (배치 or 수동 트리거) |
| FR-19 | 수집 필드: 게임명, 설명, 아이콘 URL, 평점, 다운로드 수, 카테고리, 스토어 링크 |

## 8. 데이터 모델 (주요)

### Game

| 필드 | 타입 | 설명 |
|------|------|------|
| Id | int | PK |
| Name | string | 게임명 |
| Description | string | 게임 설명 |
| IconUrl | string | 아이콘 이미지 URL |
| Rating | float | 평점 |
| DownloadCount | string | 다운로드 수 |
| Category | string | 게임 카테고리 |
| PlayStoreUrl | string | Google Play 링크 |
| AppStoreUrl | string | App Store 링크 |
| CreatedAt | datetime | 생성일 |

### HealthTag

| 필드 | 타입 | 설명 |
|------|------|------|
| Id | int | PK |
| GameId | int | FK → Game |
| Tag | string | 건강 효과 태그 |
| Confidence | float | AI 신뢰도 점수 |
| AiDescription | string | AI 설명 효과 근거 |

## 9. 구현 우선순위

### Must Have (절대 필수)
- 게임 목록 화면 (카테고리 필터 포함)
- 게임 상세 페이지
- AI 건강 효과 태깅

### Should Have (시간 되면)
- 맞춤 추천 화면
- 검색 화면

### Nice to Have (여유 있으면)
- 관리자 대시보드 별도 화면
- 게임 즐겨찾기 기능
- 건강 알림

---

## 10. 제약 사항

- **Google Play API 제한**: Google Play API 공개 제한이 있어 RapidAPI의 Google Play 관련 API 또는 Mock 데이터로 준비
- **AI 태깅 비용 최적화**: AI 태깅의 API 비용을 최적화하여 게임당 1회 분석 후 DB 캐싱하여 재사용
